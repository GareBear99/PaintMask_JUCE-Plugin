# UX Notes

## Core rule
The user should feel like they are painting music, not programming MIDI.

## Immediate first-run flow
1. Open plugin
2. See animated canvas and tool dock
3. Start audio / preview
4. Paint a stroke
5. Hear music immediately

## Do not do
- Giant generic parameter panels
- Tiny unreadable icons
- Hard lock after trial
- Corporate nag screens
- Confusing mode changes without visual feedback

## Must do
- Strong visual feedback for painted events
- Clear playhead/scan visualization
- Color meaning that feels intuitive
- Graceful Lite fallback
